Montei esse pequeno MVP com base nas solicitações do trabalho. Nessa página deixei algumas descrições para auxiliar na avaliação do código e do sistema:

Funcionalidades implementadas:
  • Adicionei tabelas extras: organizador, evento_privado e usuário.
  • Organizador, Palestrante, Participante possuem login e senha ao criarem (conectados tabela usuario e users banco).
  • Possível criar eventos e criar palestras de eventos.
  • Eventos de cascata em deletes específicos.
Dificuldades encontradas:
  • Realizar inscrição em evento privado: não consegui estruturar a lógica de participante-inscricao-evento(evento_privado)
  • Design de interação não ficou dos melhores em algumas partes
  • Algumas leituras de JSON não consegui formatar
  • Por alguns autores, não aparece a imagem do evento (todo de implementação)
Funcionalidades pedidas e onde encontrar:
  • Visualizar Imagens: consultados pelo login do admin no Gerenciador de Eventos > Visualizar.
  • Trocar Imagem: editadas pelo login do admin no Gerenciador de Eventos > Editar
  • JSON: descritas em Gerenciador de Eventos > Visualizar
TODOs e melhorias futuras:

  • Criar estrutura visual padrão
  • Adicionar verificação para todas as datas
  • Ajustar visualização de todas as páginas de evento

  • Refatorar arquitetura para não ficar espaguete
  • Organizar apresentação-negócio-persistência
  • Criar baterias de testes
  • Organizar o main para ficar mais legível
  • Permitir troca de senha para todos os usuários
  • Adicionaria try-catch e transações no banco, para garantir transações sem comprometer dados.

Em suma, o sistema está operacional com suas limitações. Participante e organizador são cadastrados manualmente ou pelo admin. Palestrante é cadastrado pelo organizador ou admin, mas cada um possui login e senha.
